# GenIoT-Optimizer — Project Description

## 1) What this project is
GenIoT-Optimizer is a full-stack AI system for simulating and optimizing IoT network performance.
It combines:
- Synthetic traffic generation (WGAN-GP)
- Anomaly detection (VAE reconstruction error)
- Policy optimization (PPO reinforcement learning)
- Real-time digital twin dashboard (FastAPI + WebSocket + React)

The goal is to improve network quality in dynamic conditions by reducing latency and energy while increasing throughput and QoS, and by detecting abnormal behavior early.

---

## 2) Core modules we built

### A. Synthetic Traffic Generator (WGAN-GP)
- File: `backend/models/wgan_gp.py`
- Purpose: Generate realistic IoT traffic windows (temporal sequence data).
- Key idea:
  - Generator maps latent noise to sequence windows.
  - Critic scores real vs generated sequences with Wasserstein objective.
  - Gradient penalty enforces Lipschitz stability.

### B. Anomaly Detection Engine (VAE)
- File: `backend/models/vae.py`
- Purpose: Detect anomalous traffic using reconstruction error.
- Key idea:
  - Encoder learns latent distribution of normal behavior.
  - Decoder reconstructs sequence.
  - High reconstruction error => likely anomaly.

### C. RL Network Optimizer (PPO)
- Files: `backend/environment/iot_network_env.py`, `backend/models/ppo_agent.py`, `backend/training/train_rl.py`
- Purpose: Learn action policies that optimize network state.
- State (observation): 12 network metrics (normalized)
- Action: 4 controls (routing, sleep duty, power, buffer tuning)
- Reward balances latency, throughput, energy, and QoS.

### D. API + Realtime Stream
- Files: `backend/api/main.py`, `backend/api/routes.py`, `backend/api/websocket.py`
- Purpose:
  - Serve model-backed REST endpoints
  - Stream live network metrics every ~500 ms over WebSocket

### E. Dashboard Frontend
- Files: `frontend/src/...`
- Stack: React + Vite + Zustand + Recharts
- Features:
  - Live metric cards
  - Real-time chart (rolling window)
  - Anomaly panel
  - WebSocket auto-reconnect behavior

---

## 3) End-to-end system flow
1. Environment produces network states over time.
2. WGAN can generate synthetic traffic for simulation/testing.
3. VAE scores anomalies from incoming sequences.
4. PPO/GenIoT policy predicts control actions.
5. FastAPI exposes APIs and pushes live metrics via WebSocket.
6. React dashboard displays live telemetry and anomaly events.

---

## 4) APIs implemented
- `POST /api/generate-traffic` → generate synthetic traffic
- `POST /api/detect-anomaly` → return anomaly score + flag
- `POST /api/optimize` → return recommended control config
- `GET /api/metrics` → current snapshot
- `GET /api/health` → service/model health
- `WS /ws/live-metrics` → realtime stream

---

## 5) Baseline comparison work
We implemented baseline evaluation to compare standard policies against GenIoT:
- RandomPolicy
- DQNBaseline (discretized wrapper over same continuous environment)
- GreedyHeuristic
- LSTMPredictor
- GANOnlyHeuristic
- GenIoT (trained PPO policy)

Metrics tracked:
- Latency (ms)
- Throughput (Mbps)
- Energy (nJ/bit)
- Anomaly F1

Outputs are saved under `results/` as table/chart artifacts.

## 9) Project scope and assumptions
- This is a simulation/optimization environment, not direct control of physical IoT devices.
- Network metrics are normalized inside the environment and converted to practical units in evaluation/output.
- Real-time dashboard values are generated by backend simulation + model logic.
